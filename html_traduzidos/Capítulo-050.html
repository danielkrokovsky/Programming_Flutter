Exemplo de uso de pacote

Agora que você sabe como instalar pacotes, verá os pacotes em ação.

Começando

Vamos construir um aplicativo simples que exibe a bateria do telefone e o estado de conectividade usando os pacotes de bateria [28] e conectividade [29] do Pub:

Para usar esses pacotes, começaremos adicionando os pacotes às dependências de nosso aplicativo, e você pode fazer isso adicionando o seguinte às dependências em pubspec.yaml :

packages / battery_connectivity / pubspec.yaml
​  conectividade: ^ 0.3.1
​  bateria: ^ 0.2.2

E importe os pacotes em nosso código adicionando ao topo do arquivo Dart principal o seguinte:

packages / battery_connectivity / lib / main.dart

​

Além dos pacotes que baixamos do Pub, também precisamos do Material Design API padrão do Flutter e da biblioteca assíncrona do Dart , que é necessária para lidar adequadamente com a natureza assíncrona dos pacotes que estamos usando.

Obtendo os dados de bateria e conectividade: programação assíncrona usando streams

Como os dados que vamos mostrar podem mudar a qualquer momento, os pacotes permitem que os dados sejam usados ​​na forma de Stream s, que permitem que seu código escute as mudanças nos dados e defina as ações que precisam ser tomadas usando os novos dados quando eles mudam.

Nesse caso, queremos alterar o estado do widget alterando as strings e ícones que descrevem a conectividade e o estado da bateria.

Podemos fazer isso usando o seguinte código:

packages / battery_connectivity / lib / main.dart
① StreamSubscription <ConnectivityResult> _connSub;
​  StreamSubscription <BatteryState> _battSub;
​ 
​  @sobrepor
​  vazio initState () {
​  Super .initState ();
​ 
② _connSub = Conectividade ()
​  .onConnectivityChanged
​  .listen ((ConnectivityResult res) async {
​  setState (() {
③ interruptor (res) {
​  caso ConnectivityResult.mobile:
​  _connectionStatus = 'Mobile Data' ;
​  _connectionIcon = Icons.sim_card;
​  _connectionColor = Colors.red;
​  quebrar ;
​  caso ConnectivityResult.wifi:
​  _connectionStatus = 'Wi-Fi' ;
​  _connectionIcon = Icons.wifi;
​  _connectionColor = Colors.blue;
​  quebrar ;
​ 
​ 
​  caso ConnectivityResult.none:
​  _connectionStatus = "Sem conexão" ;
​  _connectionIcon = Icons.not_interested;
​  _connectionColor = Colors.black;
​  }
​  });
​  });
​ 
④ _battSub = Bateria ()
​  .onBatteryStateChanged
​  .listen ((res. BatteryState) async {
​  setState (() {
​  interruptor (res) {
​  caso BatteryState.charging:
​  _batteryStatus = 'carregamento' ;
​  _batteryIcon = Icons.battery_charging_full;
​  _batteryColor = Colors.blueAccent;
​  quebrar ;
​  caso BatteryState.full:
​  _batteryStatus = 'Full' ;
​  _batteryIcon = Icons.battery_full;
​  _batteryColor = Colors.green;
​  quebrar ;
​  caso BatteryState.discharging:
​  _batteryStatus = 'em uso' ;
​  _batteryIcon = Icons.battery_alert;
​  _batteryColor = Colors.redAccent;
​  }
​  });
​  });
​  }
①

Essas duas linhas são usadas para definir as StreamSubscriptions que usaremos em nosso aplicativo. assinaturas são usadas para executar algum código quando os dados retornados por um fluxo são alterados.

②

Estamos chamando o construtor de conectividade para obter um objeto e ouvir em seu membro onConnectivityChanged Stream , que, como todos os Stream s, tem uma função de membro de escuta , que toma como argumento posicional a função de retorno de chamada a ser chamada quando a conectividade muda, o que irá geralmente é uma chamada para setState , já que na maioria dos aplicativos uma mudança em um Stream significa que a IU do aplicativo precisa ser atualizada.

③

No caso de você não estiver familiarizado com ele, a utilização de uma instrução switch é explicado no interruptor . Neste caso, estamos usando para definir diferentes strings, ícones e cores para cada conexão e status da bateria.

④

Isso é o mesmo que vimos para o status de conectividade, mas para o status da bateria.

Como esse código está fazendo muitas coisas diferentes, vamos dividir isso e explicar cada parte.

Eliminação de StreamSubscriptions

Para cada inscrição é preciso haver uma entrada para o descarte método para eliminar a subscrição quando não é mais necessário. StreamSubscription s tem um método cancel , que faz exatamente isso. Como as classes State precisam substituir o método dispose de sua superclasse (que é o método executado quando esse State não é mais necessário, que é quando o StatefulWidget não está mais sendo renderizado), elas precisam chamar super.dispose :

packages / battery_connectivity / lib / main.dart
​  @sobrepor
​  dispose () {
​  Super .Dispose ();
​  _connSub.cancel ();
​  _battSub.cancel ();
​  }

Você pode descobrir mais sobre Fluxo s em Streams e StreamSubscriptions .
