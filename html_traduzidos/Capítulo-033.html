Escrita principal

Vamos começar pela parte mais simples e menos específica de nosso aplicativo: a função principal .

Usando uma classe Wrapper

No capítulo anterior, definimos uma classe MyApp e usamos main simplesmente para executá-la, da seguinte maneira:

​  vazio principal () =>
​  runApp (MyApp ());
​ 
​  classe Meu_apl estende StatelessWidget {
​  @sobrepor
​  Construção de widget (contexto BuildContext) {
​  retorno MaterialApp (
​  // app conteúdo real e de dados
​  );
​  }
​  }

Pulando a aula de invólucro

O mesmo efeito seria alcançado simplesmente escrevendo:

​  vazio principal () {
​  runApp (MaterialApp (
​  // outros conteúdos aplicativo e dados
​  ));
​  }

Parece muito mais curto, mas em um aplicativo maior não é muito em comparação.

Se seu aplicativo estiver inteiramente contido em uma visualização, ele não comprometerá a funcionalidade do aplicativo, mas se você precisar de mais visualizações e precisar compartilhar dados, considere escrever uma classe de wrapper para seu aplicativo para manter as coisas organizadas e claras.

E, em qualquer caso, você só deve usar a segunda forma se realmente precisar manter seu código compacto ou economizar tempo para escrever um pequeno aplicativo rapidamente.

Então, vamos escrever o seguinte:

layout / calculator / lib / main.dart
​  vazio principal () => ExecutarAplicação (MyApp ());
​ 
​  classe Meu_apl estende StatelessWidget {
​  @sobrepor
​  Construção de widget (contexto BuildContext) {
​  retorno MaterialApp (
​  Título: "Flutter Calculator" ,
​  tema: ThemeData (
​  primarySwatch: Colors.blue,
​  backgroundColor: Colors.black26
​  ),
​  casa: CalculatorHomePage (título: "Flutter Calculator" ,),
​  );
​  }
​  }
