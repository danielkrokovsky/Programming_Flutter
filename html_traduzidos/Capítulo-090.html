Construções e expressões condicionais

Algumas linguagens (como Rust) tratam construções condicionais como expressões. O Dart só faz isso quando eles estão dentro de um literal de lista; em outros lugares, você precisaria usar uma expressão condicional com a sintaxe convencional do tipo C usando o operador ternário ? : .

A construção if-else

A construção if-else no Dart é muito semelhante à de muitas outras linguagens; pode ser expresso usando chaves:

​  se (condição) {
​  // As instruções para executar quando a condição for verdadeira
​  } Else {
​  // As instruções para executar quando a condição é falsa
​  }

e, se houver apenas uma instrução a ser executada para cada um, sem chaves (executando true_instruction quando a condição for verdadeira e false_instruction quando for falsa), geralmente escrita sem quebras de linha:

​  se (condição) true_instruction;
​  outra false_instruction;

mais se

Se houver uma condição para verificar dentro da cláusula else, você poderá usar a construção else if :

​  int um;
​  // (...)
​  se (a> 10) {
​  // (...)
​  } Else se (a == 10) {
​  // (...)
​  } Else {
​  // (...)
​  }

Expressões condicionais

Se você precisar testar uma condição para decidir entre dois valores para uma atribuição, uma expressão ou uma instrução de retorno, você pode usar uma expressão condicional. É uma expressão que contém o operador ternário ?: E pode ser usada da seguinte maneira:

​  var a = condição? true_value: false_value;

em que o valor de a será true_value se a condição for verdadeira e false_value se a condição for falsa . Isso é extremamente útil para substituir funções simples, como as seguintes:

​  Num max ( núm um, num b) {
​  se (b> a) retorno b;
​  else retorno a;
​  }

com linhas ainda mais simples:

​  num máximo ( num um, num b) => b> a? BA;

Interruptor

Switch é um dos recursos do Dart que pode parecer mais arcaico: funciona exatamente como C ou Java. Eles são usados ​​quando você precisa comparar uma variável a um certo número de valores constantes e não são recomendados para uso em aplicativos Flutter. Sua estrutura é a seguinte:

​  var um;
​  // (...)
​ 
​  interruptor (a) {
​  caso 1:
​  // instruções para executar se um é igual a 1
​  quebrar ;
​  Caso 2:
​  // instruções para executar se um é igual a 2
​  quebrar ;
​  padrão :
​  // instruções para executar se um não é nem igual a 1 ou 2
​  }

Se você omitir as instruções de quebra , os casos definidos após o caso disparado pelo valor da variável também serão executados. Este efeito ( fall-through ) permite que vários valores sejam tratados por um único caso :

​  interruptor (a) {
​  Caso 1: // isto é suposto cair
​  Caso 2:
​  // instruções para os valores 1 e 2
​  quebrar ;
​  Caso 3:
​  // instruções para o valor 3
​  quebrar ;
​  padrão :
​  // (...)
​  }

O fall-through obtido pela omissão do break não é permitido se você tiver instruções no caso e irá gerar um erro. Se você pretende fazer isso, deve usar a instrução continue com um rótulo:

​  interruptor (a) {
​  caso 1:
​  // instruções adicionais para 1
​  continuar caseTwo;
​  caseTwo:
​  Caso 2:
​  // instruções para 1 e 2
​  quebrar ;
​  padrão :
​  // (...)
​  }
