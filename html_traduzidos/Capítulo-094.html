Importação de masterização

A diretiva import no Dart, como é o caso em muitas outras linguagens modernas, não trata apenas da inclusão de arquivos de biblioteca externos no código como o bom e velho C #include : ela pode fazer mais do que isso para tornar nossa vida mais fácil ao lidar com com bibliotecas e pacotes.

O básico

Começaremos com o básico: importar um arquivo inteiro para o namespace atual.

Importando um arquivo local

Você pode importar outro arquivo Dart local usando um caminho relativo para ele:

​  importação 'filename.dart' ;

Importando uma Biblioteca Dart Integrada

Você pode importar uma biblioteca Dart integrada precedendo o nome da biblioteca com dart ::

​  importação 'dardo: async' ;

Importando de um pacote

Você pode importar um arquivo de um pacote usando:

​  importação 'pacote: packagename / filename.dart' ;

Que pode ser usado para importar arquivos do Flutter SDK usando flutter como o nome do pacote:

​  importação 'pacote: vibração / services.dart'

Essa sintaxe também pode ser usada para importar arquivos no diretório lib do aplicativo usando o nome do pacote do aplicativo como o nome do pacote.

Usando como para importar em um namespace

Você também pode importar em um namespace, usando a seguinte sintaxe:

​  importação 'customlib.dart' como MyCustomLib;

Para que, por exemplo, se houver uma classe CustomLibraryClass definida nesse arquivo, você pode acessá-la usando MyCustomLib.CustomLibraryClass em vez de apenas CustomLibraryClass .

Isso permite que você importe muitas bibliotecas sem sobrecarregar seu namespace principal, mantendo cada biblioteca separada, sem correr o risco de ter várias classes com o mesmo nome.

Outra grande vantagem vem quando você volta ao seu código depois de um tempo: você saberá exatamente de qual biblioteca aquela classe é, sem ter que descobrir isso primeiro.

Usando show para importar apenas algumas partes de um arquivo

import também pode ser usado para importar apenas algumas classes dentro de um arquivo.

Por exemplo, se você só precisa da CustomLibraryClass do arquivo customlib.dart e não deseja importar outras classes (talvez porque elas têm o mesmo nome de outras classes em seu namespace e você não deseja usar como e lidar com notação de ponto extra), que você pode fazer com o seguinte código:

​  importação 'customlib.dart' mostrar CustomLibraryClass;
