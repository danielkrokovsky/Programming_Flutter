A chave

A chave é usada para identificar exclusivamente um widget e é usada pela estrutura para atualizar os elementos que realmente precisam ser atualizados.

Normalmente você não usará chaves, já que o framework cuidará delas por si só, mas você pode usá-las para ter maior controle do estado e das atualizações do widget.

Além disso, existem widgets que solicitam expressamente uma chave, como o descartável que s encontrados em ListTiles descartável .

Muitas chaves para a mesma fechadura

Existem muitos tipos de Key se alguns são realmente mais úteis do que o construtor simples de Key , que toma como um argumento nomeado uma String que deve ser única dentro do pai do widget.

​  ExampleWidget (
​  chave: Chave ( "exemplo" )
​  )

GlobalKey s também existem e, devido ao seu escopo global, devem ser absolutamente únicas.

As GlobalKey s permitem que os widgets sejam movidos para serem filhos de um pai completamente diferente, preservando seu estado.

Também existem LocalKey s, que têm o mesmo escopo que Key s normais , mas têm alguns construtores diferentes:

  • O ValueKey , que assume um valor de qualquer tipo, não apenas uma String .
  • O UniqueKey , que não tem valor em tudo e é sempre único (o que usamos na descartável ListTiles ).

Usando Chaves

Na maioria das vezes, você não precisa usar as teclas ; eles só se tornam necessárias quando os widgets exigem que você defina uma chave , como aconteceu quando falamos sobre o descartável em ListTiles descartável .
