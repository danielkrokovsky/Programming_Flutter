Instalando o SDK e os plug-ins IDE

Para usar o Flutter, você precisa instalar seu SDK e, para poder continuar com sua programação de uma maneira rápida e descomplicada, você provavelmente vai querer instalar as ferramentas IDE também.

Se você preferir usar a linha de comando (talvez porque deseja usar outros IDEs ou editores de texto, talvez mais leves), haverá orientação sobre o uso do comando flutter ao longo do livro.

Instalando Flutter

O processo de instalação do Flutter SDK difere um pouco para cada sistema operacional, portanto, separarei as instruções em três seções. Pule para as instruções específicas para sua plataforma e, se desejar instalá-las, veja a parte que cobre a instalação do plug-in IDE.

No momento da escrita, a versão estável mais recente é 1.9.1 e requer um download de 495 MB no Linux (arquivo tar.xz), um download de 655 MB no Windows (arquivo zip) ou um download de 786 MB no macOS (arquivo zip).

Instalando no Linux

Em uma máquina Linux, apenas o desenvolvimento Android é compatível, portanto, instalaremos apenas o SDK do Android e o próprio SDK do Flutter. Você receberá orientação para os métodos de instalação gráfica e CLI.

Instalando o Android SDK no Linux

Se você nunca desenvolveu aplicativos Android, você precisa instalar o Android SDK, que inclui as ferramentas necessárias para construir e depurar aplicativos Android.

Para usar o Flutter, você precisa instalar o Android Studio e as ferramentas do Android SDK. O Flutter requer a instalação do Android Studio, mas você não precisa usá-lo para desenvolvimento.

Alternativas ao arquivo zip oficial

No Gentoo e Arch Linux você pode usar pacotes para instalar o Android SDK e / ou Android Studio:

  • No Gentoo, você pode instalar o pacote Android Studio [4] executando emerge --ask dev-util / android-studio ;

  • Para Arch Linux, há um pacote popular e mantido ativamente disponível no AUR chamado android-studio [5] .

Comece indo para a página de downloads na página Android Developers: [6] lá você encontrará links para o download .zip do Android Studio (cerca de 1 GB de tamanho).

Independentemente do que você escolher, pode ser necessário instalar algumas bibliotecas de 32 bits para fazer o SDK funcionar em um sistema operacional de 64 bits:

  • No Debian / Ubuntu, execute sudo apt install libc6: i386 libncurses5: i386 libstdc ++ 6: i386 lib32z1 libbz2-1.0: i386

  • No Fedora, execute sudo dnf install zlib.i686 ncurses-libs.i686 bzip2-libs.i686 .

Depois disso, estamos prontos para realmente instalar o Android SDK.

Para fazer isso usando o instalador completo do Android Studio, depois de extrair o arquivo zip que você baixou do site do Google, execute o script studio.sh contido no subdiretório bin . Isso iniciará um assistente de configuração. Depois de concluir a configuração do SDK, você pode iniciar o Android Studio executando o mesmo script studio.sh .

Instalando o Flutter SDK no Linux

Esta parte fornecerá orientação para a instalação do Linux destinada a iniciantes; se você se sentir confortável com a linha de comando, as etapas da CLI serão mais previsíveis. (Para iniciantes completos no Linux, pode ser mais fácil seguir as orientações orientadas para a GUI, já que isso geralmente é mais familiar.)

Alternativas ao Tarball

Existem alternativas para instalar o tarball oficial:

  • O Arch Linux tem um pacote mantido ativamente no AUR para instalar o Flutter. [7]

O download do Linux é um tarball de origem que também contém o script necessário para executar o comando flutter , o que significa que você só precisa extraí-lo e adicionar o subdiretório bin do tarball extraído à variável de ambiente PATH .

Antes de fazermos isso, precisamos navegar até a página do arquivo SDK [8] no site oficial do Flutter e baixar a última versão estável.

Como alternativa, em um ambiente somente CLI, você pode baixar o tarball 1.2.1 usando curl https://storage.googleapis.com/flutter_infra/releases/stable/linux/flutter_linux_v1.2.1-stable.tar.xz -o flutter_linux_v1. 2.1-stable.tar.xz .

Extraia o tarball que você acabou de baixar com qualquer ferramenta GUI de sua preferência ou executando o seguinte comando:

​  $ tar -xf flutter_linux_v1.2.1-stable.tar.xz

Agora precisamos adicionar o script executável à variável de ambiente PATH .

Antes de fazer isso, você precisa anotar o diretório onde extraiu o tarball. Ele contém um diretório flutter , dentro do qual existe um diretório bin ; precisamos saber o caminho para chegar a esse diretório bin .

Se você estiver usando a GUI, na maioria das distribuições ela está disponível navegando nas propriedades do diretório. Será algo parecido com /home/username/Downloads/flutter_linux_v1.9.1-stable/flutter/bin se você tiver optado pelas configurações padrão para cada parte do software usado nas etapas que descrevemos anteriormente. Eu sugiro mover isso para um caminho mais permanente; idealmente, um em que você se lembre de ter instalado o Flutter.

Se você trabalhou na CLI usando os comandos descritos anteriormente, navegue até o diretório de instalação do Flutter e altere o diretório de trabalho para o subdiretório flutter / bin executando:

​  $ cd flutter / bin

e obtenha o diretório de trabalho executando:

​  $ pwd

que retornará algo nas linhas de / home / username / flutter / bin (aqui você verá o caminho onde instalou o Flutter, portanto, sua milhagem pode variar significativamente).

Conheça sua Shell

Esta seção supõe que seu shell seja Bash. [9] Este é o caso da maioria das distribuições Linux (e sistemas operacionais semelhantes ao Unix em geral - algumas exceções são os BSDs, que têm tcsh, [10] ksh, [11] ou ash, [12] e o instalador do Arch Linux, que roda em Zsh [13], mas instala o Bash por padrão) e, como você precisaria instalar e configurar manualmente um diferente, provavelmente saberia adicionar um diretório ao seu PATH .

Para adicionar isso à variável de ambiente PATH , precisamos editar ~ / .bash_profile .

Para fazer isso usando a GUI, primeiro você precisa do seu gerenciador de arquivos para exibir os arquivos ocultos. Se você não conseguir encontrar um arquivo chamado .bash_profile em seu diretório inicial, será necessário alternar a opção que faz o gerenciador de arquivos mostrar arquivos ocultos e isso depende do gerenciador de arquivos que você está usando:

  • Em Nautilus (aka GNOME Files, padrão na maioria das distribuições utilizando o desktop GNOME como o Ubuntu, RHEL, Fedora e padrão SLED e Debian), você precisa pressionar Ctrl + H . Este atalho também funciona no PCManFM (parte do LXDE, como encontrado no Lubuntu), Caja (parte do Mate) e Thunar (parte do desktop XFCE, como encontrado no Xubuntu).

  • Se você estiver usando o desktop KDE (por exemplo, ao executar o Kubuntu ou ao escolhê-lo ao instalar distribuições como openSUSE ou Debian) e seu gerenciador de arquivos Dolphin padrão, use Alt + + .

Depois de localizar um arquivo chamado .bash_profile , abra-o com qualquer editor de texto e adicione a seguinte linha ao final dele, em uma nova linha:

​  exportar PATH = $ PATH: / home / nome de usuário / etc

onde você substituirá / home / username / etc pela string que você anotou anteriormente.

Se você preferir usar a linha de comando ou apenas quiser uma experiência de copiar e colar deste guia de instalação, você pode abrir uma janela de terminal ou TTY e executar o seguinte comando:

​  $ Echo "export PATH = $ PATH : / home / username / etc" >> ~ / .bash_profile

substituindo / home / username / etc pelo caminho que você encontrou anteriormente para o diretório flutter / bin do tarball extraído .

Para realmente conseguir executar o comando flutter , você precisa atualizar a configuração do seu terminal executando:

​  $ source ~ / .bash_profile

Instalando no Windows

Assim como no Linux, o desenvolvimento do Android somente para Windows é compatível, portanto, instalaremos apenas o Android SDK com o Android Studio e o próprio Flutter SDK.

Instalando o Android SDK no Windows

Tudo que você precisa fazer para instalar o Android SDK no Windows é baixar e executar o arquivo de instalação do Android Studio disponível na página oficial de download. [14] Isso também instalará o Android Studio IDE rico em recursos. Depois de instalar o Android Studio, você será guiado pela instalação do Android SDK.

Instalando o Flutter SDK no Windows

Para instalar o Flutter SDK no Windows, você precisa primeiro instalar o Git para Windows. Você pode encontrar o arquivo de instalação para ele em sua página oficial de download. [15] Durante a instalação, você precisa escolher a opção Usar Git no prompt de comando do Windows .

Depois de instalar o Git, baixe a versão mais recente do Flutter em sua página oficial de instalação. [16] Isso, ao contrário do que acontece com a maioria dos softwares disponíveis para Windows, fará o download de um arquivo zip (também conhecido como uma pasta compactada) que contém uma pasta flutuante : extraia onde quiser (esta será a pasta SDK que qualquer plugin IDE irá pedir para você entrar).

Se você deseja executar um comando no Console do Flutter, execute o script flutter.bat contido na subpasta bin .

Instalando no macOS

A vantagem do macOS é que ele também oferece suporte à criação e depuração de iOS. Para tirar proveito disso, também precisamos instalar o Xcode e o iOS SDK.

Instalando o Xcode e o iOS SDK

Para ser capaz de executar o Flutter e usá-lo para desenvolvimento iOS, você precisa baixar o Xcode [17] e o iOS SDK [18] do site oficial da Apple.

Instalando o Android SDK no macOS

Instalar o Android SDK no macOS é realmente simples: vá para a página de downloads do Android Studio [19] para baixar o arquivo .dmg que também inclui o Android Studio IDE rico em recursos, mas com muitos recursos.

Instalação do Flutter SDK no macOS

A instalação do Flutter no macOS é muito semelhante à instalação do Flutter no Linux. As principais diferenças são que o macOS usa o formato de arquivo zip em vez do formato tar.xz, mais eficiente, e não há diferenças de sistema operacional ou GUI entre as distribuições a serem enfrentadas.

Comece baixando o .zip estável mais recente do site oficial do Flutter. [20]

Descompacte-o em qualquer diretório e, em seguida, abra o diretório flutter que acabou de extrair. Imprensa ⌘ + I e tomar nota do caminho que aparece à direita do Onde: .

Agora, abra uma janela do Terminal e execute o comando:

​  $ nano ~ / .bash_profile

Depois de fazer isso, cole o seguinte na janela do terminal:

​  exportar PATH = $ PATH: / example / path / to / flutter / bin

substituindo / example / path / to / flutter / bin pelo caminho que você anotou anteriormente.

Tudo o que resta a fazer é fechar o arquivo usando Ctrl - X e confirmando que você deseja salvar o arquivo pressionando Y .

Para realmente conseguir executar o comando flutter , você precisa atualizar a configuração do seu terminal executando:

​  $ source ~ / .bash_profile

Instalando os Plugins IDE

Os plug-ins Flutter IDE para VSCode e Android Studio são instalados usando as ferramentas e técnicas de instalação canônicas para cada IDE. Eles também requerem a instalação do respectivo plugin Dart.

Se você não estiver familiarizado com esse processo, discutiremos agora como instalar o plug-in Flutter em cada IDE.

VSCode

Para instalar o plug-in Flutter para Visual Studio Code, você precisa abrir o próprio Visual Studio Code e, em seguida, usar o atalho de teclado Ctrl - Shift - P para abrir a paleta de comandos, digite “Instalar extensões” e pressione Enter .

No painel que se abre à esquerda, digite Flutter e pressione Enter .

Clique no botão Instalar na entrada Flutter na lista, isso instalará os plug-ins Flutter e Dart.

Ao contrário do plug-in Flutter para Android Studio, o plug-in Flutter para o plug-in Visual Studio Code detectará automaticamente o local onde o SDK do Flutter está instalado.

Android Studio

Para instalar o plug-in Flutter para Android Studio, você precisa navegar até Arquivo> Configurações> Plug-ins, clicar em Procurar repositórios ... , digitar Flutter e clicar em Instalar.

Para criar um projeto Flutter no Android Studio, você precisa reiniciar o Android Studio se não o fez desde a instalação do plug-in Flutter, navegue até Arquivo> Novo projeto Flutter, escolha Aplicativo Flutter e dê ao aplicativo um nome e um caminho, defina o nome do pacote e pronto.

Usando a CLI e os plug-ins

Existem duas maneiras de interagir com o Flutter SDK: usando o comando flutter CLI diretamente ou por meio dos plug-ins IDE. Os plug-ins para Android Studio e VSCode são muito diferentes de usar, pois se adaptam às convenções usuais do respectivo IDE.

Criar um projeto de aplicativo Flutter

Para criar um projeto de aplicativo Flutter, execute o comando flutter create appname ou siga o procedimento correspondente ao IDE que deseja usar para desenvolver seu aplicativo:

  • No Android Studio, clique em Arquivo> Novo> Novo Projeto Flutter ... e siga as instruções que aparecerão na tela;

  • No Visual Studio Code, pressione Ctrl + Shift + P para abrir o Menu de Comandos e digite New Project, Flutter: New Project deve aparecer como uma opção no menu, clique nele e digite um nome para seu projeto de aplicativo.

Construindo e funcionando

Você pode construir um aplicativo Flutter usando o comando flutter build quando o diretório de trabalho for o diretório raiz do projeto (onde pubspec.yaml está localizado):

  • Para construir um APK para o seu aplicativo (arquivo de instalação padrão do Android), use o apk flutter build .

  • Para construir um pacote de aplicativos iOS, use flutter build ios .

  • Para construir um Android App Bundle (a nova geração de arquivo de instalação do Android que reduz o tamanho do arquivo), use flutter build appbundle .

Se você deseja executar o aplicativo diretamente em um emulador aberto ou em um dispositivo conectado por USB (com a Depuração USB ativada nas configurações do dispositivo), use o flutter run .

Se estiver usando um IDE, você pode usar o plug-in Flutter para executar aplicativos diretamente do IDE.

Para executar um aplicativo em um emulador aberto ou dispositivo USB conectado:

  • Usando o VSCode, pressione F5 ou clique em Debug> Start Debug .

Hot Reload

O Flutter vem com um recurso que você deve conhecer de outras estruturas: hot reload. Se você não estiver familiarizado com ele, o recurso hot reload permite que você atualize a visualização atual do aplicativo com base nas alterações feitas no código sem reconstruir todo o aplicativo, o que, como qualquer pessoa que já tentou sabe, é muito doloroso quando você está fazendo várias pequenas alterações e tentando ver como elas afetam o aplicativo.

Esse recurso pode ser acessado no Android Studio e no Visual Studio Code e, na verdade, também na linha de comando quando você executa um aplicativo usando flutter run : pressionar R no terminal quando o comando estiver em execução recarregará o aplicativo.

No Visual Studio Code, você precisa encontrar um ícone como este no menu que aparece na parte superior da tela quando você inicia a depuração:

O aplicativo será recarregado imediatamente, mostrando todas as alterações feitas no código do aplicativo imediatamente no emulador ou dispositivo conectado.

Se você estiver executando uma versão recente do Android Studio e / ou Visual Studio Code, terá acesso IDE a dois recursos diferentes, mas semelhantes: recarregamento a quente com estado e reinicialização a quente. A diferença entre os dois é que o recarregamento a quente com estado (como o nome indica) preserva cada um dos objetos de estado e é mais rápido, mas não funciona com mudanças maiores que exigem que eles sejam recarregados também, e para isso há um reinício a quente .

Ao executar um aplicativo usando flutter run a partir da CLI, o reload dinâmico com estado é associado ao caractere r minúsculo , enquanto a reinicialização a quente é obtida com o caractere R maiúsculo .

Atualizando e mantendo sua instalação do Flutter

Você saberá quando precisará atualizar a instalação do Flutter SDK porque receberá um aviso sempre que criar ou executar um aplicativo.

Para atualizar a instalação do Flutter, você precisa executar o comando flutter upgrade , que verificará e fará o download de tudo o que for necessário usando o Git, portanto, você precisa ter o Git instalado.

O Flutter inclui uma ferramenta chamada Flutter Doctor, disponível usando o comando flutter doctor , que listará informações sobre a versão do Flutter instalada, o Android / iOS SDK, os plug-ins IDE e os dispositivos conectados.
