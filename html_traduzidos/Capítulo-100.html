Configuração específica da plataforma

Nesta seção, veremos como alterar o nome do aplicativo e o ícone do iniciador de um aplicativo Flutter editando a configuração específica da plataforma.

Definir um nome de aplicativo personalizado

O nome do pacote Dart (que também é o nome do aplicativo padrão) deve ser composto apenas de letras minúsculas e sublinhado, o que não é muito. É por isso que vamos definir nosso próprio nome de aplicativo nos aprofundando na configuração específica da plataforma.

Configurando o nome do aplicativo Android

O nome do aplicativo exibido no Android pode ser alterado alterando o atributo android: label da tag do aplicativo no arquivo AndroidManifest.xml localizado em appdir / android / app / src / main . Seria assim se estivéssemos configurando o aplicativo Chat on Fire que criamos no último capítulo do livro:

firebase / chatonfire2 / android / app / src / main / AndroidManifest.xml
​  <aplicação
​  android: name = "io.flutter.app.FlutterApplication"
​  android: label = "chat on Fire"
​  android: icon = "@ mipmap / ic_launcher" >

Configurando o nome do aplicativo iOS

O nome do aplicativo exibido no iOS geralmente é alterado abrindo o projeto Runner.xcworkspace Xcode no diretório ios no Xcode e alterando a configuração de nível de projeto Nome de exibição . Como alternativa, adicione uma chave chamada CFBundleDisplayName seguida por uma string ao arquivo Info.plist no diretório appdir / ios / Runner , como no exemplo a seguir:

firebase / chatonfire2 / ios / Runner / Info.plist
​  < K e y > CFBundleDisplayName < / k e y >
​  < Seqüência > Bate-papo on Fire < / string >

Configurando o ícone do iniciador

Qualquer aplicativo decente precisa de um ícone personalizado, e aplicativos Flutter certamente não fazem uma exceção a isso. Existe uma ferramenta CLI no Pub chamada flutter_launcher_icons que simplifica isso, mas você pode optar por fazer isso manualmente. Esta seção cobrirá ambos os casos.

Usando ícones do Flutter Launcher para alterar o ícone do Launcher

O pacote Ícones Iniciador vibração é o exemplo típico de uma embalagem que deve ser adicionado à dev_dependencies secção em pubspec.yaml :

firebase / chatonfire2 / pubspec.yaml
​  dev_dependencies:
​  flutter_launcher_icons:
​  flutter_test:
​  SDK: vibração

Depois de fazer isso e executar:

​  pacotes de $ flutter recebem

é hora de configurar o pacote.

Um exemplo rápido e simples

Isso pode ser feito em pubspec.yaml ou em um novo arquivo adicionando uma seção flutter_icons :

firebase / chatonfire2 / pubspec.yaml
​  flutter_icons:
​  image_path: " icons / launcher_icon.png"
​  android: true
​  ios: verdadeiro

Isso é o que você usará na maioria das vezes: o image_path será o caminho para a imagem que será usada para substituir a imagem do iniciador padrão para Android e iOS quando você executar:

​  Os pacotes $ flutter executam flutter_launcher_icons: main

Escolhendo um caminho diferente para o arquivo de configuração

Esse comando funcionará se a configuração estiver em pubspec.yaml ou em um arquivo que você criou chamado flutter_launcher_icons.yaml . Se quiser usar um arquivo personalizado com um nome diferente, você pode usar a opção -f da seguinte maneira:

​  Os pacotes $ flutter executam flutter_launcher_icons: main -f filename.yaml

Usando arquivos diferentes para Android e iOS

Em vez de fornecer um image_path para Android e iOS, você pode escolher definir um image_path_android para Android e um image_path_ios para iOS. Você pode querer fazer isso porque, ao contrário do Android, o iOS substituirá qualquer transparência por preto, o que significa que os ícones do aplicativo iOS devem preencher a imagem inteira e não ter bordas transparentes, ao contrário dos ícones do Android.

Alterar o ícone do iniciador manualmente

Em vez de usar o pacote flutter_launcher_icons , você pode querer mudar o ícone do iniciador você mesmo. Isso consome muito tempo para fazer 100% manualmente: para Android, você pode alterar o ícone do aplicativo fornecendo diferentes resoluções do ícone do iniciador para dispositivos com diferentes valores de densidade de pixel em appdir / android / app / src / main / res / mipmap- * dpi (seguindo as diretrizes oficiais do desenvolvedor Android [59] ). Um processo semelhante é necessário para os ícones do iOS para substituir aqueles no appdir / ios / Runner / Assets.xcassets / Appicon.appiconset .

Uma maneira de contornar isso é usando a ferramenta de criação de recurso de imagem do Android Studio, que irá gerar automaticamente diferentes resoluções do ícone, e abrindo Assets.xcassets em appdir / ios / Runner com Xcode e usando isso para substituir os ícones padrão.

Com isso, seu aplicativo deve estar pronto para distribuição a seus usuários, seja qual for o sistema operacional em que o telefone estiver funcionando.
