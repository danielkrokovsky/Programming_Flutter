Uma breve história da vibração

Em 2015, o Google revelou o Flutter, um novo SDK baseado na linguagem Dart, como a próxima plataforma para o desenvolvimento do Android, e em 2017 uma versão alfa (0.0.6) foi lançada ao público pela primeira vez.

No I / O 2017, o Google mostrou o uso do Flutter e seus recursos multiplataformas e continuou a promovê-lo no I / O 2018. Desde então, o Google tem investido no Flutter e recomendado como a forma como todos deveriam desenvolver aplicativos móveis.

Em dezembro de 2018, o Flutter 1.0 foi lançado e disponibilizado para que os desenvolvedores pudessem começar a usar o SDK para facilitar a criação de aplicativos.

No Google I / O 2019, o suporte do Flutter para plataformas desktop e web foi anunciado publicamente. Foram lançadas ferramentas para desenvolver aplicativos Flutter para Windows, macOS, Linux e web.

Além de ser instável e não testado, o desenvolvimento de desktop está sendo retardado pela falta de suporte a plug-ins, que é muito limitado principalmente porque, no momento em que este artigo foi escrito, o conjunto de ferramentas de plug-in ainda está em processo de desenvolvimento, o que significa que os binários para o código específico da plataforma para cada plataforma deve ser criado manualmente e vinculado editando os Makefile s fornecidos pelo Google, que podem ser encontrados no repositório GitHub dedicado ao flutter-desktop-embedding do Google . [1]

Por outro lado, o suporte da web está progredindo rapidamente e não deve levar muito mais do que a reconstrução de um projeto móvel Flutter funcional que não tem nenhum plug-in nativo ou código específico de plataforma.
