Usar interpolação de string

Neste caso, esta é uma alternativa à sintaxe que usamos anteriormente para converter um inteiro em uma string ( _counter.toString () ).

É assim que os literais de string funcionam no Dart: eles são colocados entre aspas simples ou duplas. E podemos incluir variáveis ​​em literais de string precedendo o nome da variável com um cifrão.

Se precisarmos usar interpolação de string com uma expressão ou notação de ponto (por exemplo, para usar variáveis ​​de membro de objetos ou exibir resultados de expressões), precisamos colocar o nome da variável entre colchetes. Precisamos fazer isso em breve para o título do aplicativo.

Você pode encontrar mais informações sobre strings literais e interpolação de string em Personagens e Cordas .

Faça o botão de reset

Agora que definimos um método que redefine o contador e dispara uma nova renderização, precisamos criar um botão para chamar essa função. Portanto, antes de fazermos isso, precisamos saber como fazemos isso com o Flutter.

Botões no Flutter: o FlatButton e o IconButton

Se quisermos criar rapidamente um botão no Flutter, podemos escolher entre um FlatButton ou um IconButton .

O FlatButton é ideal quando você deseja um botão que exibe texto: ele tem um atributo filho que geralmente é um widget de Texto , mas você tem a opção de defini-lo como qualquer coisa.

A razão pela qual FlatButton é usado principalmente para botões de texto é que o uso de um IconButton existe especificamente para a criação de botões baseados em ícones, permitindo ao desenvolvedor especificar um objeto Icon como seu atributo de ícone , com o botão consistindo apenas no ícone e uma pequena quantidade de preenchimento.

Um dos atributos que mostram a diferença de foco é que o FlatButton ‘s cor controles de atributos a cor do botão em si (o fundo ao seu conteúdo), enquanto o IconButton ‘s cor controle de atributo é a cor do ícone.

Um RaisedButton também existe, mas é apenas um FlatButton ligeiramente diferente .

Faça um widget personalizado: nosso próprio botão

O botão que estamos criando é RawMaterialButton , uma classe de botão básica que nos permite personalizar muito mais sua aparência do que um FlatButton mais simples .

Mais especificamente, os argumentos que estamos passando para seu construtor são:

  • onPressed , que é o mesmo que os outros onPressed atributos que vimos, e nós atribuímos OurButton ‘s onPressed atributo a ele.

  • fillColor , que é o mesmo que o atributo de cor que vimos usado para o FlatButton anteriormente neste capítulo: é a cor de fundo do botão, que será definida para o atributo backgroundColor de OurButton , para mostrar como os nomes de atributo de nosso botão personalizado podem ser diferentes quando em comparação com os do widget em que o baseamos.

  • splashColor é a cor usada para o efeito inksplash e usamos o argumento padding para fornecer uma quantidade normal (padrão) de preenchimento ao nosso botão.

  • filho é simplesmente um Texto , usando a cor especificada nos argumentos de OurButton como a cor do texto.

  • forma é a forma do botão, e estamos usando um StadiumBorder () para dar ao nosso botão os cantos arredondados que desejamos.
