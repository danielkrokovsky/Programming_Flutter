As classes no Dart funcionam muito como as classes em outras linguagens orientadas a objetos, mas aqui está a maior parte do que você precisa saber para começar.

Membros: Propriedades e Métodos

Propriedades são variáveis ​​declaradas como membros de uma classe:

​  classe usuário {
​  int id;
​  String de nome de usuário;
​  String de senha;
​  }

Métodos são funções definidas dentro de classes. Eles são definidos apenas como funções de nível superior, mas dentro do corpo de uma definição de classe:

​  classe usuário {
​  int id;
​  String de nome de usuário;
​  String de senha;
​ 
​  vazio setId ( int newId) {
​  id = newId;
​  }
​  }

Acessando membros: Notação de pontos

Os membros podem ser acessados ​​usando a notação de ponto. Se você queria para obter o ID de propriedade do usuário objeto e atribuí-lo à uma variável, você escreveria:

​  var a = user.id;

e você executaria o método setId assim:

​  user.setId (1000)

Cascade Notation

Se precisar executar vários métodos ou alterar muitas propriedades de um objeto, você pode usar a notação em cascata, que permite escrever algo assim, o que criará um usuário com o nome de usuário, senha e ID fornecidos:

​  var a = Utilizador ()
​  ..username = "YourUserName"
​  ..password = "Actually1veryComplicatedPass!"
​  ..setId (1000);

Construtores

O Dart nos permite definir apenas a assinatura do construtor e ele atribuirá automaticamente os argumentos às propriedades correspondentes e criará uma instância da classe automaticamente. Os parâmetros são definidos como qualquer parâmetro de função, mas apontando para uma propriedade da classe:

​  classe ClassExample {
​  ClassExample (
​  este .var1, // argumento posicional
​  {
​  @required este .var2, // argumentos nomeados
​  este .var3
​  }
​  );
​ 
​  int var1;
​  Cordas var2;
​  Função var3;
​  }

Você pode definir o construtor para uma classe que herda de uma superclasse da seguinte maneira:

​  classe ClassExample estende SuperClass {
​  ClassExample (parâmetros):
​  super- (superClassArguments);
​  }

Construtores Nomeados

Você também pode definir construtores nomeados , que são construtores alternativos com um nome diferente:

​  classe ClassExample {
​  ClassExample (parâmetros);
​  ClassExample.constructorName (otherParameters);
​  }

Construtores regulares

O Dart também oferece suporte ao construtor regular com seu próprio corpo, que pode, opcionalmente, aproveitar a atribuição automática de propriedade.

Aqui está um exemplo de uma classe que tem um construtor que pega um valor inteiro e atribui seu quadrado ao var1 enquanto atribui var2 e var3 diretamente:

​  classe ClassExample {
​  ClassExample ( este .var1, este .var2, este .var3);
​  ClassExample.withSqrtOfVar1 ( int var1sqrt, este .var2, este .var3) {
​  var1 = var1sqrt * var1sqrt;
​  }
​ 
​  int var1;
​  Cordas var2;
​  Função var3;
​  }

Classes abstratas

Uma classe abstrata é uma classe que você não pode instanciar: você não pode criar objetos dessa classe. Eles são declarados substituindo a palavra-chave class por classe abstrata . Classes abstratas são geralmente utilizados como mixins (cobertos em escrevendo uma classe com um Mixin ) ou interfaces (cobertos em escrevendo uma classe que implementa uma interface ).

Uma classe abstrata pode conter métodos abstratos, que são métodos que não têm uma implementação, mas também pode conter métodos regulares:

​  abstract class MyClass {
​  vazio regularMethod () {
​  // fazer alguma coisa
​  }
​ 
​  vazio abstractMethod ();
​  }

Um método abstrato pode ser implementado por uma subclasse ou uma classe que implementa a interface MyClass , conforme explicado nas seções a seguir.

Subclasses, interfaces e mixins: estende, implementa e com

As classes podem interagir umas com as outras de três maneiras: uma classe pode ser uma subclasse de outra, herdando todos os seus membros e, opcionalmente, substituindo alguns dos métodos; ele pode implementar a interface de outra classe; ou pode usar métodos definidos em outra classe (chamados de mixin).

estende

Especialmente no Flutter, estender outra classe é uma ocorrência muito comum. Fazendo isso, criamos uma subclasse, e a classe que estendemos é chamada de superclasse. Podemos decidir declarar novos métodos ou reimplementar ( substituir ) métodos existentes. Uma subclasse pode ter suas próprias variáveis ​​e usar as da superclasse acessando a própria superclasse por meio do superobjeto .

@sobrepor

Ao criar subclasses, você pode querer implementar um dos métodos da superclasse, você pode fazer isso prefixando-o com a anotação @override, como fazemos para o método de construção ao definir widgets:

​  classe MyWidget estende StatelessWidget {
​  // (...)
​ 
​  @sobrepor
​  Construção de widget (contexto BuildContext) {
​  // (...)
​  }
​  }

o método também pode ser definido como:

​  @override Widget build (contexto) {
​  // (...)
​  }

Às vezes você pode querer chamar o método correspondente da superclasse adicionando super.methodName no início da definição do novo método de modo que o novo método atua como um complemento, não um substituto, como nós temos que fazer quando redefinir o Estado de classe initState quando escrevendo StatefulWidget s:

​  classe _MyWidgetState estende Estado <StatefulWidget> {
​  // (...)
​ 
​  @sobrepor
​  vazio initState () {
​  Super .initState ();
​  // nossas próprias instruções
​  }
​ 
​  // (...)
​  }

Temos que fazer isso para a classe State porque ela foi definida na biblioteca de widgets Flutter com a anotação @mustCallSuper , e podemos fazer o mesmo caso desejemos definir classes para as quais achamos que as subclasses serão criadas e uma certa sobreposição O método deve chamar o método sobrescrito da superclasse.

Escrever uma aula com um Mixin

Um mixin é uma classe que contém métodos a serem usados ​​por outras classes, sem exigir que essas classes herdem do mixin diretamente, estendendo-o e tornando-se uma subclasse dele. Um mixin pode ser definido simplesmente como uma classe e ser usado declarando qualquer classe que queira usá-lo ( aplicando a mixagem) com o mixin, como no exemplo a seguir:

​  // o mixin
​  classe MyMixin {
​  Cordas myMethod () => "Hi do mixin" ;
​  }
​ 
​  classe MyClass com MyMixin {
​ 
​  }
​ 
​  var obj = MyClass ();
​ 
​  assert (obj.myMethod () == "Hi do mixin" );

Escrevendo uma classe que implementa uma interface

Uma interface é uma classe que define apenas o tipo de retorno e os argumentos de seus métodos, exigindo que outras classes implementem esses métodos. Adequadamente, isso é feito usando a palavra-chave implements :

​  abstract class MyInterface {
​  abstrato vazio myMethod ();
​  }
​  classe MyClass implementa MyInterface {
​  vazio myMethod () {
​  // fazer alguma coisa
​  }
​  }
​ 
​  classe MyOtherClass implementos MyInterface {
​  vazio myMethod () {
​  // fazer outra coisa
​  }
​  }

Depois de fazer isso, você pode instanciar e usar MyClass e MyOtherClass , e pode usá-los sabendo que implementam todos os métodos descritos em MyInterface .
