Layout Vertical

Existem duas opções disponíveis quando o desenvolvedor precisa dispor os itens verticalmente, dependendo do que for necessário: a coluna e o ListView .

A coluna

A coluna é usada para criar uma pilha vertical de widgets e não é rolável. É usado para exibir itens próximos uns dos outros verticalmente e não para criar uma lista de itens.

Como você usa isso

Você pode fazer uma coluna simples desta maneira:

​  Coluna(
​  filhos: <Widget> [
​  // widgets separados por vírgulas
​  ],
​  )

Opções de widget de layout

Você só precisa colocar os widgets dentro dos colchetes do Widget coleção literal (mais sobre literais de coleta e listas em listas e literais de cobrança ) e os widgets serão mostrados em uma pilha vertical, mas há um monte de opções para customize nos argumentos do construtor, como o mainAxisAlignment , que define como os widgets filhos devem ser alinhados ao longo do eixo principal (no caso de uma coluna é o eixo vertical), e pode ser um dos seguintes:

  • MainAxisAlignment.start , MainAxisAlignment.center ou MainAxisAlignment.end , que colocará os widgets o mais próximo possível, respectivamente, do início, meio ou final do eixo principal.

  • MainAxisAlignment.spaceAround , que usa metade do espaço disponível para separar os filhos uns dos outros e a outra metade para separar o primeiro e o último deles do início e do final da coluna.

  • MainAxisAlignment.spaceBetween , que usa o espaço para tornar os filhos o mais distantes possível entre si.

  • MainAxisAlignment.spaceEvenly , que separa cada filho do próximo tanto quanto o início da Coluna para o primeiro e o último para o final da 'Coluna.

Estreitamente relacionado está o mainAxisSize , que pode ser MainAxisSize.max ou MainAxisSize.min .

Há também crossAxisAlignment , que define como os widgets filhos devem ser alinhados ao longo do eixo cruzado (no caso de uma coluna , é o eixo horizontal) e pode ser um dos seguintes:

  • O CrossAxisAlignment.center padrão , que alinha os widgets para que seu centro fique alinhado.

  • CrossAxisAlignment.start e CrossAxisAlignment.end que, respectivamente, alinham os widgets no início ou no final do eixo cruzado.

  • CrossAxisAlignment.stretch , o que faz com que os widgets ocupem o máximo de espaço possível no eixo cruzado.

  • CrossAxisAlignment.baseline , especialmente útil com Row 's, que alinha a linha de base dos filhos.

Para que você usa isso

Você pode usar uma coluna simplesmente, como no capítulo anterior, para mostrar dois widgets um em cima do outro, como este:

​  corpo: coluna (

​  mainAxisAlignment: MainAxisAlignment.spaceBetween,

​  Ícone (Icons.arrow_upward),
​  Texto ( "Isso vai estar no meio" ),
​  Texto ( "Este será na parte inferior" )
​  ],
​  ),
​  )

mas provavelmente você desejará usá-lo com mais frequência para agrupar widgets, talvez formando um widget novo e mais complexo:

O ListView

O ListView é usado para criar uma lista rolável de widgets.

É usado quando há necessidade de criar longas listas de itens semelhantes (ou iguais) e pode carregar dinamicamente outros elementos quando são muitos (ou ilimitados).

A lista pode ser expressa de várias maneiras, como veremos agora.

Como você usa isso

Existem vários construtores que você pode usar para criar um ListView .

O construtor padrão funciona como o construtor da coluna : você só precisa fornecer uma lista de widgets como filhos :

​  Exibição de lista(
​  filhos: <Widget> [
​  Texto ( "1" ),
​  Texto ( "2" ),
​  Texto ( "3" ),
​  ...
​  ],
​  )

Este exemplo mostra a vantagem de usar outro construtor, o construtor ListView.builder , que é usado da seguinte maneira:

​  ListView.builder (
​  itemCount: 499,
​  itemBuilder: (contexto BuildContext, int i) =>
​  Texto ( " $ {i + 1} " ),
​  )

Isso faz o mesmo e cria uma lista de 499 widgets de texto com os números de 1 a 500.

Mas esse construtor também pode ser útil quando cada elemento tem algo diferente - por exemplo, se cada Texto precisa de uma string diferente e não relacionada.

Depois de definir algo como:

​  const cordas = [
​  "O primeiro widget" ,
​  "A Segunda Widget" ,
​  "A Terceira Widget" ,
​  "The Last Widget"
​  ]

ou gerar uma lista de lista usando o que é descrito em listas de Geração .

Você pode criar um ListView como este:

​  ListView.builder (
​  itemCount: strings.length,
​  itemBuilder: (contexto BuildContext, int i) =>
​  Texto (strings [i]),
​  )

Há também ListView.separated , que permite adicionar um widget separador, neste caso, outro Text para simplificar:

​  ListView.separated (
​  itemCount: 500,
​  itemBuilder: (contexto, i) =>
​  Texto ( " $ i " , estilo: Theme.of (contexto) .textTheme.display1,),
​  separatorBuilder: (contexto, i) =>
​  i% 3 == 0? Divider (): Padding (padding: EdgeInsets.all (0.5),),
​  )

shrinkWrap , quando definido como falso (padrão), fará com que a lista ocupe o máximo de espaço disponível no eixo principal. Se, em vez disso, você definir como verdadeiro, o tamanho da grade se adaptará ao tamanho necessário para exibir seus filhos .

Personalizando o Divisor

O divisor pode ser personalizado para se adequar à aparência geral, comportamento e necessidades do seu aplicativo:

  • Você pode definir a opção de cor para outra cor.

  • Você pode definir sua opção de altura fornecendo um número duplo para tornar o espaço maior ou menor.

  • Você pode definir sua opção de recuo fornecendo um número duplo para fazer a linha começar em uma posição diferente ao longo do eixo horizontal.

O ListTile é o widget típico usado com ListView se é muito comum em aplicativos de bate-papo: a linha típica feita de uma imagem de visualização (geralmente redonda), à esquerda e à direita um título e um subtítulo opcional.

ListTile é muito comum e útil e muito fácil de implementar no Flutter:

Use um CircleAvatar para arredondar Widgets

A alternativa é usar um CircleAvatar , que simplifica o processo quando pode ser usado.

Um CircleAvatar é comparável a um contêiner de círculo pré-fabricado e é otimizado para arredondar imagens, mas também pode ser usado como um contêiner.

Por exemplo, você poderia replicar exatamente o exemplo anterior do Container de uma maneira mais simples, como esta:

O ExpansionTile é um ListTile expansível : é como um ListTile quando é renderizado pela primeira vez, mas você pode, sem sair da visualização atual, tornar um ExpansionTile mais alto e mostrar mais widgets, como mais texto ou botões e recursos adicionais.

ListTiles descartáveis

Um uso muito comum para ListTile s é como widgets descartáveis, o que significa que eles podem ser excluídos com um gesto de deslizar. Um widget ganha essa capacidade ao ser agrupado em um widget Dismissível .

Para ignorar, é necessário fornecer uma chave . Não é estritamente necessário entender Key s usar descartável s, mas uma explicação mais aprofundada será fornecido em The Key .

O widget Dismissible é usado da seguinte maneira:

​  Dismissível (
​  fundo: Container (color: Colors.green),
​  chave: chave (string),
​  filho: ListTile (
​  título: Texto (string),
​  ),
​  )

onde o fundo é o que é mostrado atrás do bloco quando ele é deslizado pelo usuário. Configurá-lo como um contêiner permite escolher a cor de fundo.

Uma implementação completa de Dismissible dentro de um ListView seria:

​  ListView.builder (
​  itemCount: 20,
​  itemBuilder: (contexto, i) => Dismissível (
​  fundo: Container (
​  color: Colors.red,
​  filho: Row (
​  mainAxisAlignment: MainAxisAlignment.spaceBetween,
​  crianças: [
​  Ícone (Icons.clear, color: Colors.white,),
​  Ícone (Icons.clear, color: Colors.white,),
​  ],
​  ),
​  ),
​  chave: UniqueKey (),
​  filho: ListTile (
​  líder: CircleAvatar (
​  backgroundImage: NetworkImage ( "https://pragprog.com/favicon.png" ),

​  ),
​  ),
