E se tentarmos dividir por 0?

Ficamos preocupados com a má formação da expressão (erros de sintaxe), mas não nos preocupamos com erros matemáticos.

Ao escrever uma calculadora na maioria das linguagens de programação (especialmente se estiver usando aritmética de inteiros, o que não é o nosso caso), fazer um aplicativo como o nosso (que não calcula a raiz quadrada ou funções trigonométricas) nos deixa expostos a algo que pode torná-lo crash: dividindo por 0.

O Dart, ao usar aritmética de ponto flutuante como no nosso caso, é inteligente o suficiente para exibir a palavra Infinito em vez de um resultado; isso é bom porque não faz com que o aplicativo trave, mas alguns podem argumentar que não é realmente a resposta correta na maioria dos casos: conforme o divisor se aproxima de 0, o quociente se aproxima do infinito, mas a divisão por zero em si é impossível, portanto, você pode optar por exibir um erro se estiver fazendo uma calculadora e se sentir inclinado a fazê-lo.

Se for esse o caso, você precisaria adicionar uma condição a cada cálculo em Calculation.getResult para verificar se há uma tentativa de divisão por 0, lançar uma exceção e capturá-la no método de chamada.

Lançamento e captura de exceções

Esta seção pode ser considerado uma antecipação de jogar e pegar exceções , não estamos indo para cobrir o tema em detalhe e se você não está acostumado a esse tipo de recurso de linguagem de programação, você deve considerar a leitura que o primeiro, e depois de ler este seção.

Pegue a exceção

As exceções no Dart são detectadas usando:

​  tente {
​  code_that_might_generate_exceptions ();
​  } Prendedor (exceção) {
​  imprimir ( " $ exceção " );
​  }

Este código captura qualquer tipo de exceção; se você quisesse capturar apenas certos tipos de exceções, você usaria:

​  tente {
​  code_that_might_generate_exceptions ();
​  } Em ExceptionClass captura (exceção) {
​  imprimir ( " $ exceção " );
​  }

E se você não precisa usar o valor da exceção, você deixaria de fora a parte catch (exception) :

​  tente {
​  code_that_might_generate_exceptions ();
​  } em ExceptionClass {
​  imprimir ( "você fez algo ruim" );
​  }

No caso específico, o código que pode gerar exceções é a chamada e atribuição de função _str = _calculation.getResult (). ToString () e queremos definir _str para algo como Você não deve dividir por 0 .
